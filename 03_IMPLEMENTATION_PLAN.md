## **MVP 구현 단계 목록**

에이전트가 작업한다는 전제이면, **기능 단위보다 “독립 검증 가능한 개발 단위”**로 끊는 것이 중요합니다.
추천 순서는 아래와 같습니다.

### **1단계. 프로젝트 뼈대 구성**

* 백엔드 기본 구조
* 프론트 기본 구조
* DB 스키마 초안
* 공통 설정, 환경변수, 실행 스크립트
* 로컬 실행/배포 방식 정리

**완료 기준:** 앱이 로컬에서 실행되고, 기본 화면/API 헬스체크가 동작함.

### **2단계. 전략 도메인 구현**

* 전략 생성/수정/복제/삭제
* 전략 버전 관리
* 전략 상태(초안/백테스트완료/실행중지) 관리
* 유니버스(대상 종목군) 관리

**완료 기준:** 전략과 종목군을 저장·조회·수정할 수 있음.

### **3단계. 전략 편집기**

* 빌더형 전략 입력 UI/API
* 코드형 전략 입력 UI/API
* 전략 정의 검증
* 전략 미리보기 또는 파싱 결과 확인

**완료 기준:** 사용자가 전략을 실제 실행 가능한 형태로 저장할 수 있음.

### **4단계. 백테스트 엔진**

* 백테스트 실행 API
* 기간/비용/슬리피지 설정
* 거래 결과 계산
* 결과 요약 및 거래 내역 화면

**완료 기준:** 전략별 백테스트 결과를 보고 실행 여부를 판단할 수 있음.

### **5단계. 자동 실행 엔진**

* 장중/종가 실행 스케줄
* 전략별 ON/OFF
* 시그널 생성
* 실행 로그 기록

**완료 기준:** 설정된 전략이 주기적으로 실행되고 시그널이 남음.

### **6단계. 주문 엔진 + 프리체크**

* 주문 생성
* 중복 주문 방지
* 잔고/장시간/수량/가격 검증
* 모의/실전 주문 분리

**완료 기준:** 유효한 시그널만 실제 주문 요청으로 연결됨.

### **7단계. 포지션/체결/주문 추적**

* 주문 상태 저장
* 체결 내역 반영
* 현재 포지션 계산
* 주문/체결/포지션 조회 화면

**완료 기준:** 지금 무엇을 주문했고 무엇을 보유 중인지 확인 가능함.

### **8단계. 리스크 통제**

* 종목당 투자 한도
* 전략당 최대 노출
* 동시 보유 수 제한
* 일일 손실 한도
* 킬 스위치

**완료 기준:** 전략 오류가 나도 시스템 차원의 최소 안전장치가 동작함.

### **9단계. 인증정보/계좌 연결**

* 한투 API 인증 처리
* 모의/실전 모드 분리
* 계좌 연결 상태 확인 API
* 설정 화면 기본 구현

**완료 기준:** 계좌 연결 테스트와 연결 상태 표시가 가능함.

### **10단계. 운영 대시보드/로그**

* 대시보드 요약 화면
* 최근 오류/실행 로그
* 상태 모니터링
* 외부 접속 시 확인용 핵심 화면 정리

**완료 기준:** 대시보드만 봐도 전체 운영 상태를 파악할 수 있음.

### **11단계. 안정화**

* 통합 테스트
* 재시작 복구
* 실패/예외 처리 보강
* 실전 전 모의운영 점검

**완료 기준:** 모의환경에서 일정 기간 안정적으로 동작함.

## **권장 원칙**

에이전트 작업이면 특히 아래 원칙이 좋습니다.

* **한 단계마다 머지 가능한 상태로 끝내기**
* **각 단계마다 완료 기준을 문서화하기**
* **백엔드/프론트 동시 진행보다 도메인 우선으로 진행하기**
* **실전 주문은 인증정보/계좌 연결과 리스크 통제 이후에만 연결하기**

한 줄로 정리하면,
**“전략 저장 → 백테스트 → 실행 → 주문 → 추적 → 통제” 순서로 쪼개는 것이 가장 안전합니다.**
