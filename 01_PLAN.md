## **앱 1차 기획서**

### **1. 목적**

본 앱은 **한국투자증권 Open API를 기반으로, 사용자가 직접 전략을 만들고 백테스트한 뒤 자동매매에 적용하는 1인용 운영 도구**를 만드는 것을 목표로 한다. 한투 GitHub 저장소는 공식적으로 **샘플 코드 저장소**이며, `strategy_builder`와 `backtester`, 그리고 상품별 API 예제를 제공하므로, 이를 **참조해 실사용용 앱을 새로 구성**하는 방향을 전제로 한다. ([GitHub][1])

### **2. 제품 정의**

이 앱은 **데모용 전략 빌더**가 아니라, 아래 흐름을 한곳에서 다루는 **개인용 자동매매 콘솔**이다.

**전략 작성 → 백테스트 → 전략 버전 확정 → 자동 실행 → 주문/체결/포지션 추적 → 위험 통제**

한투 저장소도 `strategy_builder → .kis.yaml → backtester → BUY/SELL/HOLD → KIS Open API` 흐름을 제시하므로, 1차 제품은 이 파이프라인을 실사용 기준으로 재구성하는 형태가 적절하다. ([GitHub][1])

### **3. 1차 범위**

1차 버전은 **국내 시장 우선**으로 구현한다.
이유는 한투 저장소 전체에는 `overseas_stock` 예제가 포함되어 있어 해외주식 API 자체는 존재하지만, 저장소의 `backtester`와 전략/리포트 설명은 국내 종목과 국내 중심 흐름을 강하게 전제하고 있기 때문이다. 따라서 **미국 시장은 2차 확장 범위**로 두는 것이 현실적이다. ([GitHub][1])

### **4. 핵심 기능**

#### **4-1. 전략 작성**

전략은 두 방식으로 지원한다.

* **빌더형 전략**

  * 가격, 거래량, 이동평균, RSI 등 조건 조합
  * AND / OR 기반 진입·청산 조건 작성
* **코드형 전략**

  * 빌더로 표현하기 어려운 고급 규칙 직접 작성

한투 저장소의 `strategy_builder`도 **비주얼 UI 기반 전략 설계**와 **BUY/SELL/HOLD 시그널 생성**, 그리고 다수의 기술지표·프리셋 전략을 제공하므로, 이 방향은 실제 구현 가능성이 높다. ([GitHub][1])

#### **4-2. 백테스트**

전략은 반드시 백테스트를 거친 뒤 실행 대상으로 승격한다.

* 기간별 수익률
* 최대낙폭(MDD)
* 승률, 손익비
* 거래별 진입/청산 기록
* 수수료/슬리피지 반영
* 전략 버전별 결과 비교

한투 저장소의 `backtester`는 **과거 검증**, **파라미터 최적화**, **HTML 리포트**를 제공하는 방향으로 설명되어 있어, 1차 앱도 이 수준을 기준선으로 삼는 것이 적절하다. ([GitHub][1])

#### **4-3. 자동 실행**

전략은 앱에서 실행 단위로 관리한다.

* 장중 실행 / 종가 실행 선택
* 전략별 ON/OFF
* 모의투자 / 실전투자 전환
* 실행 주기 설정
* 적용 종목군 선택

한투 저장소는 **실전/모의투자 환경 전환**, **웹소켓 연결 설정**, **시그널 기반 매수/매도** 구조를 제공하므로, 자동 실행 계층을 새 앱에서 안정적으로 재구성하는 것이 핵심이다. ([GitHub][1])

### **5. 반드시 포함할 운영 기능**

이 부분은 이전 초안보다 더 중요하다. 자동매매의 기초는 전략 자체보다 **통제 장치**이기 때문이다.

* **유니버스 관리**: 어떤 종목군에 전략을 적용할지 관리
* **자금 배분 규칙**: 종목당 투자한도, 전략당 최대 노출, 동시 보유 수 제한
* **프리체크(주문 전 검사)**: 장 시간, 잔고, 중복 주문, 비정상 수량/가격 검증
* **주문 생명주기 기록**: 주문 생성, 접수, 체결, 취소, 실패 이력 저장
* **포지션 추적**: 현재 보유 수량, 평균단가, 평가손익
* **긴급 중지**: 전체 전략 또는 개별 전략 즉시 중단

한투 저장소는 샘플 코드와 기본 파이프라인을 제공하지만, 프로그램 손해에 대해 책임지지 않는다고 명시하고 있으므로, **실사용 앱에서는 이 운영 통제 계층을 별도로 강화**해야 한다. ([GitHub][1])

### **6. 화면 구성**

1차 화면은 아래 정도면 충분하다.

* **대시보드**: 실행 중 전략, 포지션, 손익, 최근 오류
* **전략 목록**: 생성, 복제, 버전 관리, ON/OFF
* **전략 편집**: 빌더형 / 코드형
* **백테스트**: 기간·비용 설정, 결과 확인
* **주문/체결**: 주문 상태, 실패 사유, 체결 이력
* **설정**: 계좌 연결, 실행 모드, 알림, 원격 접속 설정

### **7. 제외 범위**

아래는 1차에서 제외한다.

* 미국 시장 완전 지원
* 다중 사용자
* 다중 브로커
* 포트폴리오 최적화
* 몬테카를로, 워크포워드 같은 연구용 고급 기능
* 모바일 앱 전용 화면

이 항목들은 필요할 수 있으나, 한투 저장소가 지금 직접 제공하는 핵심은 **샘플 API + 전략 설계 + 백테스트 + 주문 연결**이므로, 1차 제품 범위를 넘는다. ([GitHub][1])

### **8. 성공 기준**

이 앱이 성공했다고 볼 기준은 아래와 같다.

* 사용자가 전략을 **직접 작성**할 수 있다.
* 전략을 **백테스트 후 버전 고정**할 수 있다.
* 고정된 전략을 **장중 또는 종가 기준으로 자동 실행**할 수 있다.
* 주문, 체결, 포지션, 오류를 **한 화면에서 추적**할 수 있다.
* 이상 상황에서 **즉시 중지**할 수 있다.

### **9. 한 줄 정의**

**한투 Open API 샘플 저장소를 참조해, 국내 시장 우선의 1인용 자동매매 운영 앱을 구축한다. 사용자는 빌더형 또는 코드형으로 전략을 작성하고, 백테스트 후 확정된 전략을 장중 또는 종가 기준으로 실행하며, 앱은 유니버스 관리·자금 배분·주문 추적·리스크 통제를 제공한다.**
