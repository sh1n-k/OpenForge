## **MVP 기능 목록 문서**

### **문서 목적**

본 문서는 **개인용 자동매매 앱의 1차 MVP(Minimum Viable Product, 최소 기능 제품)** 범위를 정의한다.
목표는 **한국투자증권 Open API를 기반으로, 사용자가 전략을 작성하고 백테스트한 뒤 실제 자동매매에 적용할 수 있는 최소한의 실사용 제품**을 만드는 것이다.

MVP는 기능을 많이 넣는 것이 아니라, **실제로 돌아가고**, **실수에 강하며**, **다음 확장에 무리가 없는 구조**를 만드는 데 목적이 있다.

---

## **1. MVP 목표**

MVP의 핵심 목표는 아래 5가지다.

* 사용자가 **전략을 직접 만들 수 있다**
* 전략을 **백테스트로 검증할 수 있다**
* 검증된 전략을 **자동 실행할 수 있다**
* 주문/체결/포지션 상태를 **추적할 수 있다**
* 문제가 생기면 **즉시 중지하고 원인을 확인할 수 있다**

---

## **2. MVP 범위**

### **포함 범위**

* **국내 시장 우선 지원**
* **1인 사용자 전용**
* **로컬 실행 + 외부 안전 접속**
* **장중 실행 / 종가 실행 선택 가능**
* **빌더형 전략 + 코드형 전략**
* **기본 백테스트**
* **모의/실전 실행 전환**
* **주문/체결/포지션 추적**
* **기본 리스크 통제**

### **제외 범위**

* 미국 시장 완전 지원
* 다중 사용자/권한 관리
* 다중 브로커 연동
* 포트폴리오 최적화
* 고급 연구 기능(워크포워드, 몬테카를로 등)
* 모바일 앱 전용 UI
* 복잡한 통계 대시보드

---

## **3. MVP 핵심 기능 목록**

## **3-1. 인증 및 기본 환경 설정**

### **기능**

* 한국투자증권 API 연결 정보 등록
* 모의투자 / 실전투자 모드 선택
* 앱 기본 실행 환경 설정
* 원격 접속 설정 확인

### **세부 항목**

* API Key / Secret 저장
* 계좌 정보 등록
* 모의/실전 플래그 저장
* 시장 운영 기본 설정 저장
* 시스템 상태 점검 기능

### **완료 기준**

* 사용자가 앱 설치 후 **계좌 연결을 완료**할 수 있어야 한다.
* 연결 상태를 화면에서 **성공/실패로 확인**할 수 있어야 한다.

---

## **3-2. 전략 관리**

### **기능**

* 전략 생성
* 전략 수정
* 전략 복제
* 전략 저장
* 전략 ON/OFF
* 전략 버전 관리

### **세부 항목**

* 전략 이름, 설명 입력
* 전략 유형 선택

  * 빌더형
  * 코드형
* 전략 상태 구분

  * 초안
  * 백테스트 완료
  * 실행중
  * 중지
* 전략 버전 저장 및 조회

### **완료 기준**

* 사용자가 전략을 **저장하고 다시 불러올 수 있어야 한다.**
* 실행 중인 전략과 수정 중인 전략을 **구분할 수 있어야 한다.**

---

## **3-3. 빌더형 전략 작성**

### **기능**

* 조건 조합형 전략 작성 UI 제공

### **세부 항목**

* 진입 조건 작성
* 청산 조건 작성
* AND / OR 조합
* 기본 비교 연산 지원

  * 초과
  * 미만
  * 이상
  * 이하
  * 교차 상향
  * 교차 하향
* 기본 입력 변수 지원

  * 종가
  * 시가
  * 고가
  * 저가
  * 거래량
* 기본 지표 지원

  * 이동평균선
  * RSI
  * MACD
  * 거래량 평균
  * 최고가/최저가 기준 조건

### **완료 기준**

* 사용자가 화면에서 조건을 조합해 **기본 전략을 만들 수 있어야 한다.**
* 작성한 조건을 시스템이 **실행 가능한 전략 정의로 변환**할 수 있어야 한다.

---

## **3-4. 코드형 전략 작성**

### **기능**

* 고급 사용자를 위한 코드 입력 방식 제공

### **세부 항목**

* 코드 편집기 제공
* 전략 인터페이스 템플릿 제공
* 입력 데이터 형식 고정
* 시그널 출력 형식 고정
* 문법 오류 및 기본 검증 제공

### **완료 기준**

* 사용자가 빌더로 표현하기 어려운 전략을 **직접 코드로 작성**할 수 있어야 한다.
* 실행 전 **기본 검증 실패 여부**를 알려줄 수 있어야 한다.

---

## **3-5. 유니버스 관리**

### **기능**

* 전략이 적용될 종목 범위를 관리

### **세부 항목**

* 단일 종목 선택
* 여러 종목 리스트 선택
* 사용자 정의 관심종목 그룹 저장
* 전략별 적용 대상 설정

### **완료 기준**

* 사용자가 전략마다 **어떤 종목에 적용할지 명확히 설정**할 수 있어야 한다.

---

## **3-6. 백테스트**

### **기능**

* 전략 검증용 기본 백테스트 제공

### **세부 항목**

* 기간 선택
* 종목/종목군 선택
* 초기 자본 설정
* 매매 수수료 반영
* 슬리피지 설정
* 전략 실행 결과 계산
* 결과 요약 제공

### **결과 지표**

* 총 수익률
* 최대낙폭(MDD)
* 승률
* 거래 횟수
* 평균 손익
* 손익비
* 기간별 손익 곡선

### **거래 결과**

* 진입 시점
* 청산 시점
* 진입가
* 청산가
* 손익

### **완료 기준**

* 사용자가 전략 실행 전 **최소한의 성과 검증**을 할 수 있어야 한다.
* 백테스트 결과를 보고 **전략 적용 여부를 판단**할 수 있어야 한다.

---

## **3-7. 자동 실행 엔진**

### **기능**

* 전략을 주기적으로 실행하고 시그널을 생성

### **세부 항목**

* 장중 실행 여부 설정
* 종가 실행 여부 설정
* 실행 주기 설정
* 전략별 스케줄 관리
* 전략별 활성화/비활성화
* 모의/실전 모드 분기

### **완료 기준**

* 사용자가 특정 전략을 **자동 실행 상태로 전환**할 수 있어야 한다.
* 설정된 주기대로 전략이 실행되고 **시그널이 기록**되어야 한다.

---

## **3-8. 주문 엔진**

### **기능**

* 시그널을 실제 주문 요청으로 연결

### **세부 항목**

* 매수/매도 주문 생성
* 시장가/지정가 기본 지원
* 주문 전 검증
* 중복 주문 방지
* 동일 전략의 반복 진입 방지
* 주문 실패 처리
* 모의/실전 분리 처리

### **완료 기준**

* 조건 충족 시 주문이 **정상 생성 및 전송**되어야 한다.
* 잘못된 주문은 **사전에 차단**되어야 한다.

---

## **3-9. 리스크 통제**

### **기능**

* 전략 또는 시스템의 잘못된 동작을 제한

### **세부 항목**

* 전략별 1회 주문 한도
* 종목당 최대 투자 한도
* 전략당 최대 노출 한도
* 동시 보유 종목 수 제한
* 일일 손실 한도
* 긴급 중지(Kill Switch)

### **완료 기준**

* 전략이 공격적으로 작성되어도, 시스템 차원에서 **최소한의 안전장치가 동작**해야 한다.
* 사용자가 버튼 한 번으로 **전체 자동매매를 중지**할 수 있어야 한다.

---

## **3-10. 주문/체결/포지션 추적**

### **기능**

* 자동매매 실행 결과를 운영 관점에서 추적

### **세부 항목**

* 주문 목록 조회
* 주문 상태 조회

  * 생성
  * 전송
  * 접수
  * 체결
  * 실패
  * 취소
* 체결 내역 조회
* 현재 포지션 조회
* 평균단가 및 평가손익 계산
* 전략별 보유 현황 연결

### **완료 기준**

* 사용자가 현재 **무슨 주문이 나갔고**, **무엇을 들고 있으며**, **상태가 어떤지**를 확인할 수 있어야 한다.

---

## **3-11. 운영 대시보드**

### **기능**

* 현재 운영 상태를 한 화면에서 확인

### **표시 항목**

* 실행 중 전략 수
* 전략별 상태
* 금일 주문 수
* 최근 체결 내역
* 현재 포지션
* 금일 손익
* 최근 오류
* 긴급 중지 버튼

### **완료 기준**

* 사용자가 대시보드만 봐도 **전체 상태를 빠르게 파악**할 수 있어야 한다.

---

## **3-12. 로그 및 오류 추적**

### **기능**

* 실행 이력과 오류를 남기고 확인

### **세부 항목**

* 전략 실행 로그
* 시그널 생성 로그
* 주문 요청 로그
* API 실패 로그
* 오류 메시지 기록
* 최근 오류 조회

### **완료 기준**

* 문제 발생 시 사용자가 **원인을 추적할 수 있는 최소 기록**이 남아 있어야 한다.

## **5. MVP 비기능 요구사항**

### **안정성**

* 앱 재시작 후 핵심 상태 복구 가능
* 주문/체결/포지션 데이터 영속 저장
* 외부 API 실패 시 재시도 가능
* 비정상 종료 후 로그 확인 가능

### **보안**

* 외부 접속 시 인증 필요
* 민감 정보 안전 저장
* 실전 모드 진입 시 보호 장치 필요

### **운영성**

* 로컬 장기 실행 가능
* 외부에서 상태 조회 가능
* 긴급 중지 가능

## **7. MVP 완료 정의**

아래 조건을 만족하면 MVP 완료로 본다.

* 사용자가 전략을 **직접 작성**할 수 있다.
* 전략을 **백테스트**할 수 있다.
* 전략을 **자동 실행**할 수 있다.
* 주문/체결/포지션 상태를 **추적**할 수 있다.
* 위험 상황에서 **즉시 중지**할 수 있다.
* 외부에서 접속해 **상태 확인과 제어**가 가능하다.

## **8. 한 줄 요약**

**MVP는 “전략 작성, 백테스트, 자동 실행, 주문 추적, 위험 통제”를 하나로 묶은 개인용 자동매매 운영 앱의 최소 실사용 버전이다.**
